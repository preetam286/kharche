# Budget Planner — Project Structure

## Overview

Small, modular React app for splitting group expenses. Components are organized under `src/` with separate style files and a single hook for state

## Directory layout

```
src/
├── components/           # Reusable React components
│   ├── Header.jsx
│   ├── UserManagement.jsx
│   ├── AddExpense.jsx
│   ├── EditExpenseModal.jsx
│   ├── Settlement.jsx
│   └── ExpenseHistory.jsx
│
├── styles/               # Component-scoped CSS (Tailwind @apply)
│   ├── common.css
│   ├── App.css
│   ├── Header.css
│   ├── UserManagement.css
│   ├── AddExpense.css
│   ├── EditExpenseModal.css
│   ├── Settlement.css
│   └── ExpenseHistory.css
│
├── hooks/                # Custom hooks
│   └── useExpenseState.js
│
├── utils/                # Pure utility functions
│   └── settlementCalculator.js
│
├── assets/               # Static assets (images, icons)
├── App.jsx
├── main.jsx
└── index.css             # Tailwind entry
```

## Key components

- `Header` — App header and icon
- `UserManagement` — Add/remove users
- `AddExpense` — Create a new expense (checkbox grid for participants)
- `EditExpenseModal` — Modal for editing an existing expense with validation
- `ExpenseHistory` — Table of expenses (shows excluded users) with edit/delete actions
- `Settlement` — Displays simplified settlement plan

## State & utilities

- `useExpenseState` centralizes users, expenses and form state and exposes handlers (add/update/remove)
- `settlementCalculator` computes net balances and a minimal set of transactions to settle debts

## Styling

- Uses Tailwind via `@apply` in component CSS files for consistent utility-driven styles.
- Each component has a dedicated CSS file following BEM-like class names.

## Build artifacts & git

- `dist/` is generated by the build and should not be committed — it is listed in `.gitignore`.

## Running the project

```bash
# Install dependencies
npm install

# Start dev server
npm run dev

# Build for production
npm run build

# Lint
npm run lint

## Tests

Unit and integration tests use Vitest and Testing Library. Run:

```bash
npm test
```

To run tests in watch mode:

```bash
npm run test:watch
```

Manual test scripts are available under the `test/scripts/` folder for quick ad-hoc checks.

## CI

A GitHub Actions workflow is included at `.github/workflows/ci.yml` to run tests on push and pull requests.
```

## Notes

- The app is designed for clarity and easy extension: adding features (categories, persistence) is straightforward by extending the hook and components.
- Styling and component layouts prioritize responsive behavior and accessibility.
